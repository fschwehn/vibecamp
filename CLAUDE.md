# vibecamp — working notes

A tiny pocket arcade: self-contained, mobile-first games served from GitHub
Pages. No build step, no dependencies — each game is a single HTML file. See
`README.md` for structure and per-game details.

## Workflow

- **Commit directly to `main`.** Don't develop on feature branches or open pull
  requests for changes here — commit to `main` and push. GitHub Pages deploys
  from `main`, so pushing `main` is what triggers the release.
